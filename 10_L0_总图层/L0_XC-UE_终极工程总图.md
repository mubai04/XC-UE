## 1. 定位

XC-UE 是 Xuke · Chuangjie Ultimate Engine。

中文定位：

小说工业化创作终极执行引擎。

L0 总母图只负责系统导航。

它不直接写正文。
它不直接判断章节能不能发布。
它不直接替代 Cursor 执行协议。
它不直接替代 L1、L1.5、L2、L3 文档。
它不直接替代正文真源。

L0 的任务只有四个：

1. 统一系统入口
2. 定义模块边界
3. 指向下层真源文件
4. 防止图片、规则、执行协议混用

## 3. 系统目标

核心目标：

让创作、检测、发布、反馈、学习形成可持续增益。

最高指标：

```text
UEG > 1
```

解释：

```text
UEG = 单位时间有效价值增益
```

UEG > 1：系统值得继续运行。
UEG ≤ 1：系统需要修正、降级、冻结或换项目。

## 14. 当前验收优先级

当前系统验收优先级为：

1. 正文能稳定产出
2. 章节留存变高
3. 项目系统乱不乱
4. AI味道降低
5. 长期复用

解释：

系统首先服务正文稳定产出。
其次服务读者留存。
再次控制系统混乱度。
然后降低 AI 味。
最后考虑长期复用。

不得为了长期复用牺牲当前正文产出。
不得为了系统完整牺牲章节留存。
不得为了理论完整牺牲执行效率。

禁止把待建文件写成已启用文件。

禁止在 L0 中展开 L1、L2、L3 的内部细则。

## 16. 真源规则

Markdown 是真源。

图片是导航。

Mermaid / SVG 是结构表达。

Cursor 执行协议是 L3。

正文真源只在正文目录。

任何图片中的规则，如与 Markdown 冲突，以 Markdown 为准。

任何执行结果，如与发布锁冲突，以发布锁为准。

任何对话建议，如与已落盘 Markdown 冲突，以 Markdown 为准。

任何待建文件，不得被当作已启用文件使用。
