
## 2. 能力定位

L2-06 是 XC-UE 的系统一致性与状态管理专题能力。

它回答的问题是：

```text
系统如何长期不崩？
```

更精确地说：

```text
世界、角色、事件、变化、价值、演化等对象，如何在长篇运行、能力调用、检测修复、发布反馈中不失忆、不污染、不漂移、不自相矛盾？
```

一句话定位：

```text
L2-06 不是总控层，而是五大能力长期运行的一致性底座。
```

## 4. L2-06 负责什么 / 不负责什么

L2-06 负责维护：

```text
状态一致性
历史一致性
因果一致性
设定一致性
信息一致性
关系一致性
结构一致性
选择一致性
反馈一致性
真源一致性
```

L2-06 不直接生成：

```text
世界规则 / 角色人格 / 叙事结构 / 文风表达 / 市场爽点 / 章节正文 / 大纲剧情 / 平台发布稿
```

L2-06 不直接判断：

```text
主线是否好看 / 句子是否好听 / 角色是否有魅力 / 设定是否新奇 / 读者是否付费
```

L2-06 只判断：

```text
这些对象的状态是否保存、历史是否继承、约束是否仍有效、对象之间是否冲突、修复结果是否回写。
```

## 5. 与其他 L2 的边界

### 5.1 与 L2-01 叙事结构能力

```text
L2-01 修“这一段结构怎么成立”。
L2-06 管“这个结构状态后面不能丢”。
```

L2-01 负责路径剪枝动作；L2-06 负责已剪枝路径是否回潮、结构阶段是否长期继承、前文结果是否限制后文选择。

### 5.2 与 L2-02 文风语言能力

```text
语言有没有触点 → L2-02。
触点是否与系统状态一致 → L2-06。
```

L2-02 负责信息增量、语境锁定、语言节奏波形、现实锚点、AI味抑制；L2-06 负责这些表达是否与世界状态、角色状态、关系状态、历史状态冲突。

### 5.3 与 L2-03 角色心理能力

```text
角色为什么这样行动 → L2-03。
角色过去的状态是否还影响现在 → L2-06。
```

L2-03 负责人格、欲望、执念、情绪迁移；L2-06 负责目标、伤势、债务、知情范围、关系变化是否继承。

### 5.4 与 L2-04 创意设定能力

```text
规则如何设计 → L2-04。
规则是否前后一致 → L2-06。
```

L2-04 负责规则系统、符号系统、差异设计、世界运行后果；L2-06 负责规则版本、世界状态、设定污染、规则后果是否一致。

### 5.5 与 L2-05 市场体验能力

```text
读者为什么继续 → L2-05。
反馈如何进入下一轮系统 → L2-06。
```

L2-05 负责爽点、留存、传播、付费、章末追读；L2-06 负责反馈记录、发布结果入库、失败断点回写、修复结果回原闸门复验。

### 5.6 与 L1.5 的派单边界

L2-06 发现越界问题时，不得直接调用其他 L2。

```text
只能输出“疑似越界 / 建议回 L1.5 重路由”。
正式派单由 L1.5 完成。
```

## 6. 核心定义

### 6.1 系统一致性

```text
世界、角色、事件、变化、价值、演化、信息、关系、结构、选择、反馈，在长期运行中能够共同成立。
```

### 6.2 状态管理

```text
判断哪些信息已经成为系统事实，哪些仍是可能性，哪些已经失效，哪些必须限制后续选择。
```

### 6.3 约束管理

```text
保存和执行限制条件，让角色、世界、事件、选择和能力不能随意漂移。
```

### 6.4 历史管理

```text
让已经发生的事持续影响后续系统。
```

### 6.5 因果演化管理

```text
让行为、后果、状态变化、下一次选择之间形成连续演化。
```

### 6.6 信息管理

```text
管理谁知道、谁不知道、读者知道到什么程度、误导信息是否仍有效。
```

### 6.7 关系管理

```text
管理人物、势力、利益、权力、债务、承诺和敌友状态的变化。
```

### 6.8 结构管理

```text
管理主线、副线、伏笔线、压力线、阶段问题和收束点的长期状态。
```

### 6.9 可能性管理

```text
记录哪些路径仍开放，哪些路径已剪枝，哪些路径暂存，哪些路径禁止回潮。
```

### 6.10 选择管理

```text
记录哪些选择已经发生，选择删除了什么可能性，又制造了什么新约束。
```

### 6.11 反馈回写管理

```text
将测试、发布、读者反馈、失败断点、修复结果写回正确系统位置。
```

## 9. 真源优先级

```text
P0 当前正式 Markdown 真源
P1 已人工确认的项目真源图
P2 已确认 L0 / L1 / L1.5 / L2-00 工程文件
P3 人工转译后的外部资料规则
P4 测试记录与失败断点
P5 派生展示图 / 说明图
P6 临时实验文件
P7 废弃章节与旧系统文件
```

规则：

```text
低优先级不得覆盖高优先级。
派生图不得覆盖 Markdown 真源。
图片可作为上游真源或导航，但不能自动覆盖规则文本。
旧章节实验不得参与正式 L2 定义。
对话建议不得覆盖已落盘 Markdown。
当图片与 Markdown 冲突时，以 Markdown 为准。
```

## 10. 触发条件

出现以下情况，必须进入 L2-06：

```text
1. 同一事实出现两个版本
2. 角色状态与历史冲突
3. 伏笔状态不明
4. 信息知情范围不明
5. 已剪枝路径重新出现
6. 约束失效导致选择无代价
7. 世界规则前后冲突
8. 文件真源冲突
9. 图片与 Markdown 冲突
10. 章节实验污染正式能力层
11. 多章连续性出现漂移
12. 反馈结果无法写回系统
13. 发布锁与功能锁混用
14. 旧系统文件影响新系统判断
15. 当前系统阶段不明
16. 测试、生产、发布、复盘混用
17. 状态已变化但后文仍按旧状态运行
18. 选择已经发生但被删除的可能性重新出现
19. 失败断点没有回写到对应能力层
```

L2-06 不是常驻全管层。只有出现系统一致性、状态继承、真源边界、阶段混用、反馈回写问题时，才调用 L2-06。

## 13. 失败类型库

### F01 真源污染

不该进入正式工程区的文件、图片、章节实验、旧资料或临时补丁影响正式判断。

修复：执行真源清洗，恢复真源优先级。

### F02 阶段状态混用

生产、测试、发布、复盘、能力建设等阶段混用。

修复：标记当前阶段，只允许执行该阶段动作。

### F03 系统一致性失败

世界、角色、事件、变化、价值、演化无法同时成立。

修复：执行系统一致性复核。

### F04 状态管理失败

系统无法保存或继承当前状态。

修复：建立状态快照。

### F05 约束管理失败

系统没有保存限制条件，导致可能性无限膨胀。

修复：建立约束表。

### F06 历史管理失败

已发生事实未被后续系统继承。

修复：建立历史索引。

### F07 因果演化管理失败

行为、后果、状态变化、下一次选择之间没有连续演化。

修复：重建因果演化链。

### F08 信息显隐管理失败

公开信息、隐藏信息、误导信息、角色知情范围混乱。

修复：建立信息显隐表。

### F09 关系管理失败

人物、势力、利益、权力、债务关系没有被动态继承。

修复：建立关系状态表。

### F10 结构状态管理失败

主线、副线、伏笔线、压力线、阶段问题没有被统一管理。

修复：建立结构状态表。

### F11 可能性管理失败

开放路径、已剪枝路径、暂存路径、禁止路径未被记录。

修复：建立可能性剪枝表。

### F12 选择管理失败

选择没有记录已删除的可能性、新产生的约束和后果。

修复：建立选择表。

### F13 反馈回写失败

检测、发布、读者反馈、失败断点、修复结果没有写回正确系统位置。

修复：建立反馈回写记录。

### F14 验收不可判 / 疑似越界

输入不足以判断系统一致性失败，或主问题不属于 L2-06。

修复：输出“需要补输入”或“建议回 L1.5 重路由”。

## 15. 回流验收

常见回流位置：

```text
L1-00：接口与真源问题
L1-01：内部创作成立问题
L1-02：读者投入状态问题
L1-03：发布锁与发布状态问题
L1.5：路由复核
```

验收问题：

```text
1. 当前真源是否唯一？
2. 当前系统阶段是否明确？
3. 当前状态是否可追溯？
4. 已发生历史是否被继承？
5. 约束是否有效限制可能性？
6. 因果是否连续演化？
7. 信息显隐范围是否清楚？
8. 关系变化是否有历史依据？
9. 结构状态是否可追踪？
10. 已剪枝路径是否被禁止回潮？
11. 选择是否删除某些可能性并制造新约束？
12. 反馈是否写回正确位置？
13. 是否存在低优先级资料覆盖高优先级真源？
14. 是否存在疑似越界、需要回 L1.5 重路由的问题？
```
