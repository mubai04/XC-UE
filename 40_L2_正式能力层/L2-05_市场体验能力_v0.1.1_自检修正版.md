## 0. 自检修正摘要

v0.1 的主方向成立：L2-05 应围绕 **单位时间情绪增益 UEG** 处理市场与用户体验问题。  
但 v0.1 存在五个需要修正的点：

1. **“唯一核心指标”表述过强。**  
   UEG 可以作为 L2-05 内部归一化指标，但不能宣称市场现实只有一个指标。题材、封面、更新、平台流量、价格、读者圈层也会影响结果。L2-05 只处理“内容体验如何转化为用户行为倾向”。

2. **与 L1-02 / L1-03 的边界不够硬。**  
   L1-02 / L1-03 负责判断读者投入意愿和发布锁是否通过；L2-05 不能替它们判定作品能否发布，只能在被路由后输出修复动作单。

3. **缺少“时间窗口”概念。**  
   用户不是按整本书判断，而是在开头、段落、章节、章末等短窗口内不断判断“继续读是否值”。因此 L2-05 必须先标注验收窗口。

4. **付费 / ARPPU 等外部业务指标边界不够清。**  
   L2-05 可处理“付费意愿 / 持续消费意愿”的内容信号，不处理真实平台收入、投流、渠道分发、广告、封面点击率等外部运营问题。

5. **失败类型顺序需要重排。**  
   v0.1 把“UEG 不足”放在 P2，但它过于总括。v0.1.1 将失败链改为：用户收益承诺不清 → 即时信号弱 → 预测模型无法建立 → UEG 不足 → 四因子短板 → 行为映射失败。

## 1. 层级定位

L2-05 是 **市场体验能力层**。

它不写正文，不替代 L1 / L1.5 判断，不直接决定作品能否发布，也不做外部市场运营。  
它只负责把“读者不投、不开头、不追读、不分享、不付费、弃书”的问题，转译成可仲裁的失败类型、可执行的修复动作，以及可回流验收的工程条件。

一句话定位：

> L2-05 负责提高内容在指定时间窗口内的单位时间情绪增益，使用户产生继续投入、传播、付费或持续消费的行为倾向。

## 3. 根部因果链

市场与用户体验问题的根部链条是：

```text
认知资源有限
↓
内容供给过剩
↓
用户必须做选择
↓
用户必须快速决策
↓
用户只能使用当前信号预测未来体验
↓
即时情绪变化成为最低成本、最早可感知、最可预测的信号
↓
单位时间情绪增益 UEG 成为 L2-05 的归一化指标
↓
UEG 影响用户的投入 / 退出 / 传播 / 付费倾向
```

根本判断：

> 在时间稀缺、内容过剩的环境里，用户不是先理性评估作品完整价值，而是用当前窗口内的即时情绪变化预测未来体验，并据此分配认知资源。

## 4. 核心指标：UEG

UEG：Unit Emotion Gain per time，单位时间情绪增益。

定义：

```text
UEG = 单位时间内用户获得的有效情绪变化总量
```

工程公式：

```text
UEG = 情绪强度 × 变化幅度 × 变化频率 ÷ 时间成本
```

四个因子：

1. 情绪强度：单次冲击有多强。
2. 变化幅度：每次变化有多大。
3. 变化频率：多久发生一次有效变化。
4. 时间成本：用户为了获得变化需要付出的阅读时间、认知负荷与信息消化成本。

修正版边界：

> UEG 不是现实市场的唯一指标，而是 L2-05 内部用于统一衡量用户体验收益的工程指标。

UEG 越高，用户越容易觉得“值得继续投入”。  
UEG 越低，用户越容易退出、弃书或忽略。  
但最终真实市场表现还受平台、题材、封面、更新时间、渠道分发等外部因素影响。

## 5. 时间窗口模型

L2-05 每次修复必须先标注时间窗口。

常用窗口：

1. 开头窗口：前 30 秒 / 前 300 字 / 第一屏。
2. 段落窗口：一个阅读段内是否有情绪变化。
3. 章节窗口：一章内是否形成持续收益。
4. 章末窗口：是否产生下一章预期。
5. 多章窗口：3–5 章内是否形成追读理由。
6. 付费窗口：付费点前是否形成足够收益承诺。

窗口不同，修复目标不同。

```text
开头窗口：看不看
段落窗口：累不累
章节窗口：爽不爽
章末窗口：追不追
多章窗口：留不留
付费窗口：买不买
```

## 6. 用户认知与决策模型

用户的投入 / 退出模型如下：

```text
建立预测模型
↓
接收当前内容信号
↓
产生即时情绪变化
↓
预测未来体验
↓
决策：继续 or 退出
↓
行为结果：投入 / 退出 / 传播 / 付费
```

用户不是看完全部内容再判断，而是在每一个短时间窗口内不断问：

```text
继续读，未来会不会获得更多情绪收益？
```

若答案为是，用户继续。  
若答案为否，用户退出。

## 7. 市场行为映射

UEG 驱动四类用户行为倾向。

### 7.1 投入 / 继续看

条件：

```text
当前 UEG ≥ 用户预期阈值
```

结果：

```text
继续投入时间与注意力
```

指标倾向：

- 追更意愿
- 完读意愿
- 继续阅读意愿

### 7.2 退出 / 弃书

条件：

```text
当前 UEG < 用户预期阈值
```

结果：

```text
立即或逐渐流失
```

指标倾向：

- 弃书风险
- 流失风险
- 开头跳出风险

### 7.3 传播 / 分享

条件：

```text
UEG 超出预期，且形成可复述信号
```

结果：

```text
主动推荐、评论、讨论或转述
```

指标倾向：

- 分享冲动
- 评论冲动
- 讨论点密度
- 梗点 / 反差点 / 名场面

### 7.4 付费 / 持续消费

条件：

```text
UEG 高、稳定，且未来收益承诺明确
```

结果：

```text
愿意付费获取后续体验
```

指标倾向：

- 付费意愿
- 订阅意愿
- 追更消费意愿
- 时间换收益的心理成立

注意：L2-05 只处理内容侧“行为倾向”，不处理真实收入、投流、渠道分发与平台商业策略。

## 8. UEG 工程拆解

### 8.1 情绪强度

回答：

```text
单次冲击是否够强？
```

组成：

- 冲突强度
- 反差强度
- 价值受威胁强度
- 欲望满足程度
- 支配感 / 爽感强度

### 8.2 变化幅度

回答：

```text
每次变化是否够大？
```

组成：

- 预期被颠覆程度
- 信息量差异
- 认知模型崩塌 / 重建程度
- 局势变化程度
- 价值波动程度

### 8.3 变化频率

回答：

```text
多久发生一次有效变化？
```

组成：

- 节奏密度
- 事件密度
- 爽点密度
- 悬念更新密度
- 预测更新频率

### 8.4 时间成本

回答：

```text
用户付出多少时间与认知负担？
```

组成：

- 阅读时间
- 认知负荷
- 信息消化成本
- 无效铺垫成本
- 理解门槛
- 设定解释成本

## 10. 内容生产工程

提高 UEG 的内容生产链：

```text
需求洞察
↓
收益承诺
↓
即时信号
↓
情绪设计
↓
事件设计
↓
节奏控制
↓
信息增量
↓
反馈验证
↓
持续迭代
```

### 10.1 需求洞察

回答：用户想获得什么？

可能包括：

- 爽感
- 新奇
- 反差
- 安全感
- 支配感
- 认同感
- 解压
- 期待兑现

### 10.2 收益承诺

回答：继续读，用户预期会得到什么？

必须明确：

- 当前承诺
- 下一步承诺
- 未兑现奖励
- 未来冲突
- 未来反转

### 10.3 即时信号

回答：用户在当前窗口是否快速感到“后面值得看”？

可用信号：

- 冲突信号
- 风险信号
- 奖励信号
- 反差信号
- 异常信号
- 悬念信号
- 爽点信号

### 10.4 情绪设计

回答：本窗口设计哪条情绪变化？

必须明确：

- 起始情绪
- 中段变化
- 峰值情绪
- 余波
- 下一步期待

### 10.5 事件设计

回答：用什么事件制造情绪变化？

事件必须至少改变一项：

- 局势
- 预测
- 关系
- 风险
- 利益
- 价值排序

### 10.6 节奏控制

回答：变化频率是否足够？

目标：

- 避免长时间无情绪变化
- 避免连续解释
- 避免无效铺垫
- 保证短窗口内有可感知收益

### 10.7 信息增量

回答：每段是否提供新信号？

信息必须服务于：

- 预测更新
- 悬念增强
- 局势变化
- 角色选择
- 价值波动

### 10.8 反馈验证

回答：UEG 是否真的提高？

验证对象：

- 开头是否抓人
- 是否有弃读点
- 是否有追读理由
- 是否有分享理由
- 是否有付费理由
- 是否存在高成本低收益段

## 11. 主失败类型（P1-P12）

### P1 用户收益承诺不清

表现：用户不知道继续读会获得什么情绪收益。

### P2 即时信号弱

表现：当前窗口无法快速证明后续体验值得投入。

### P3 预测模型无法建立

表现：读者不知道该期待什么，也不知道接下来会如何变强。

### P4 单位时间情绪增益不足

表现：读了不少内容，但有效情绪变化很少。

### P5 情绪强度不足

表现：冲突、反差、价值威胁、欲望满足、支配感偏弱。

### P6 变化幅度不足

表现：局势、信息、预期、认知模型没有明显变化。

### P7 变化频率不足

表现：有效变化间隔太长，读者等待成本过高。

### P8 时间成本过高

表现：阅读费力、解释过多、信息消化成本高、认知负荷大。

### P9 期待升级失败

表现：读完本段 / 本章后，未来体验预期没有变强。

### P10 行为映射失败

表现：内容无法推动投入、退出拦截、传播或付费倾向。

### P11 反馈验证缺失

表现：没有检查 UEG 是否提高，只凭主观感觉判断。

### P12 验收不可判 / 疑似越界

表现：输入材料不足，或问题实质属于 L2-01 / L2-02 / L2-03 / L2-04 / L2-06，需要回 L1.5 重路由。

- 加强冲突

- 增加反差
- 提高欲望满足
- 提升支配感

### A6 变化幅度扩大

扩大每次变化的幅度。

操作：

- 改变局势
- 颠覆预期
- 增加信息差
- 改变人物关系
- 改变风险等级
- 改变价值排序

### A7 变化频率压缩

缩短有效变化间隔。

操作：

- 删除无效解释
- 压缩铺垫
- 提高事件密度
- 提高爽点 / 悬念更新频率
- 减少长时间静态信息段

### A8 时间成本降低

降低用户阅读负担。

操作：

- 降低认知负荷
- 减少设定说明
- 减少抽象解释
- 增强场景可感知性
- 减少无效句

注意：具体句级语言修复应回 L2-02；L2-05 只判断“时间成本”是否压低。

### A9 预测模型建立

让用户知道继续读会获得什么。

操作：

- 建立短期目标
- 建立未兑现承诺
- 建立未来冲突
- 建立奖励预期
- 建立反转预期

### A10 期待升级链

让每一次事件都提高下一次期待。

链条：

```text
当前变化
↓
未来可能更大变化
↓

```

## 13. 与其他 L2 的边界

### 13.1 与 L2-01 叙事结构能力

- 结构链断、因果不收束、章节结构不成立 → L2-01
- 用户投入收益低、单位时间情绪变化不足 → L2-05

### 13.2 与 L2-02 文风语言能力

- 语言重复、AI 味、信息增量句级不足 → L2-02
- 阅读成本高导致 UEG 下降 → L2-05 可提出“降低时间成本”，但具体语言修复回 L2-02

### 13.3 与 L2-03 角色心理能力

- 角色动机、心理迁移、隐藏目的不成立 → L2-03
- 情绪收益与用户行为映射不成立 → L2-05

### 13.4 与 L2-04 创意设定能力

- 世界中心价值、约束边界、压缩入口、有效偏离不成立 → L2-04
- 设定是否转化为用户投入、传播、付费信号 → L2-05

### 13.5 与 L2-06 系统一致性能力

- 状态继承、历史、约束、因果演化断裂 → L2-06
- 长期 UEG 是否可持续、是否导致追读 / 付费 → L2-05

### 13.6 与外部运营层

- 封面、书名、简介、投流、推荐机制、平台分发 → 外部运营 / 发布策略，不归 L2-05 当前正文工程范围
- 内容内部是否形成可投入、可传播、可付费信号 → L2-05

### 13.7 与 L1 / L1.5

- L1 判断问题是否存在。
- L1.5 判断派给哪个能力层。
- L2-05 只在被路由后执行市场体验修复。
- L2-05 发现越界，只能建议回 L1.5 重路由。

## 14. 验收十四问

1. 当前修复窗口是什么？
2. 用户当前想获得什么情绪收益？
3. 继续读的收益承诺是否清楚？
4. 内容是否在短时间内提供即时情绪变化？
5. 开头 / 当前窗口是否给出足够强的未来体验信号？
6. 是否建立了用户可下注的预测模型？
7. 情绪强度是否足够？
8. 变化幅度是否足够？
9. 变化频率是否足够？
10. 时间成本是否过高？
11. 用户是否能判断继续读会获得更大收益？
12. 是否存在明确投入理由、传播理由或付费理由？
13. 是否存在明显退出风险？
14. 是否存在疑似越界、需要回 L1.5 重路由的问题？
