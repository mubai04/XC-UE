## 2. 能力定位

L2-02 是 XC-UE 的文风语言专题能力。

它负责处理文本表达中的语言组织失败。

它回答的问题是：

```text
故事如何被说出来？
语言如何在最短有效表达中持续提供认知变化、语境变化、情绪变化与预测变化？
```

L2-02 的核心职责：

```text
识别语言失败
↓
仲裁主失败
↓
输出语言修复作业
↓
回流 L1 / L1.5 验收
```

L2-02 不负责：

```text
叙事结构修复
角色心理建模
创意设定生成
世界观规则设计
市场卖点评估
爽点强度评估
IP核心符号设计
物理系统真实性验证
正文直接生成
单章轻验收记录
```

## 3. 三层边界

文风语言能力必须放在小说三层系统中理解。

```text
世界层：写什么
叙事层：怎么发生
文风层：怎么说出来
```

对应能力层：

```text
世界层 → L2-04 创意设定能力
叙事层 → L2-01 叙事结构能力
文风层 → L2-02 文风语言能力
```

L2-02 只处理“怎么说出来”。

因此：

```text
规则、符号、差异、传播
→ 主要归 L2-04

条件链、因果链、主线、结构节奏、心理映射
→ 主要归 L2-01

输入组织、语境锁定、认知压缩、语言节奏波形、现实锚点、局部符号、AI味抑制
→ 归 L2-02
```

禁止把世界层问题、叙事层问题、市场层问题混入 L2-02 主体。

## 4. 与其他 L2 的边界

### 4.1 与 L2-01 叙事结构能力的边界

L2-01 的节奏是结构节奏压缩：

```text
事件密度
信息顺序
结构延迟
因果压缩
爆点位置
压力释放
阶段推进速度
```

L2-02 的节奏是语言节奏波形：

```text
句长变化
停顿
断句
句群密度
语言压缩
表达释放
语气波动
```

边界句：

```text
L2-01 修“事情何时发生、何时爆、何时延迟”。
L2-02 修“句子如何推进、如何停顿、如何压缩”。
```

### 4.2 与 L2-03 角色心理能力的边界

L2-02 只处理语言中的角色声音呈现：

```text
称谓
语气
词汇选择
句式习惯
沉默与打断
说话时机
```

L2-03 处理角色心理本体：

```text
人格模型
欲望轴
执念
情绪迁移
关系位置
长期行为一致性
```

边界句：

```text
人物为什么这样想 → L2-03。
人物这种状态如何在语言里露出来 → L2-02。
```

### 4.3 与 L2-04 创意设定能力的边界

L2-02 处理局部语言符号触发：

```text
局部物件
重复变义的句子
场景记忆点
段内象征节点
```

L2-04 处理 IP / 世界层符号系统：

```text
世界观核心符号
规则符号
传播符号
可复用设定锚点
IP识别物
```

边界句：

```text
局部表达记忆点 → L2-02。
世界/IP符号系统 → L2-04。
```

### 4.4 与 L2-05 市场体验能力的边界

L2-02 处理语言预测收益：

```text
读完这一句后，读者对语境、人物、风险、下一句走向的判断是否改变。
```

L2-05 处理市场收益：

```text
留存
传播
付费
爽点兑现
章末追读
平台推荐表现
```

边界句：

```text
一句话是否让读者多预测一步 → L2-02。
一章是否让读者继续追 / 付费 / 传播 → L2-05。
```

### 4.5 与 L2-06 系统一致性与状态管理能力的边界

L2-02 处理语言现实锚点：

```text
触感
重量
温度
气味
空间限制
动作代价
物体状态
```

L2-06 处理系统一致性与状态继承：

```text
状态是否保存
历史是否继承
规则是否冲突
真源是否污染
已剪枝路径是否回潮
```

边界句：

```text
语言是否有可感知触点 → L2-02。
这个触点是否与世界状态、历史状态、规则状态一致 → L2-06。
```

### 4.6 与 L1.5 的派单边界

L2-02 发现越界问题时，不得直接调用其他 L2。

正确动作是：

```text
输出“疑似越界 / 建议回 L1.5 重路由”
由 L1.5 决定是否派往 L2-01 / L2-03 / L2-04 / L2-05 / L2-06。
```

L2-02 可以给出建议目标，但不能越权派单。

## 5. 核心定义

### 5.1 文风不是辞藻

文风不是“词更漂亮”。

文风是：

```text
单位语言
对读者认知、情绪、注意力、语境判断和预测模型
产生稳定增量的表达组织系统。
```

### 5.2 文风语言本体

```text
文风语言能力
=
信息增量
+
语境锁定
+
认知压缩
+
语言节奏波形
+
现实锚点
+
局部符号触发
+
语言预测收益
+
AI味抑制
```

### 5.3 一句话定义

```text
好文风 = 在最短有效语言内，持续提供认知变化、语境变化、情绪变化和预测变化。
```

### 5.4 L2-02 的工程定义

```text
L2-02 文风语言能力
=
把原始文本或语言问题包转译为可检测的语言失败，
通过信息增量、语境锁定、认知压缩、语言节奏波形、现实锚点与局部符号触发，
降低AI味，
提升语言预测收益，
并输出可回流验收的语言修复作业。
```

## 6. 文风语言生成闭环

L2-02 的主运行链为：

```text
原始文本 / 语言问题包
↓
输入组织
↓
信息增量判断
↓
语境锁定
↓
认知压缩
↓
语言节奏波形
↓
现实锚点
↓
局部符号触发
↓
语言预测收益
↓
AI味抑制
↓
语言体验成立
```

### 6.1 原始文本 / 语言问题包

输入对象：

```text
字词
句子
句群
段落
场景表达
多章语言
```

L2-02 不直接把“原始正文”改成“更好正文”。

正文必须先被 L1 / L1.5 抽象为语言失败包。

### 6.2 输入组织

回答：

```text
这段语言服务谁？
发生在什么场景？
当前压力是什么？
人物关系是什么？
读者应该获取什么？
```

失败风险：

```text
语言漂浮
谁都能说
换场景也成立
语境无约束
```

### 6.3 信息增量判断

回答：

```text
这句话让读者新知道了什么？
如果删除它，损失什么？
```

失败风险：

```text
重复
啰嗦
同义反复
无增量解释
字数增长但判断不变
```

### 6.4 语境锁定

回答：

```text
这句话为什么只能出现在这个人物、这个场景、这个关系、这个时刻？
```

失败风险：

```text
通用模板
旁白平均化
人物声音相同
缺少权力关系和场景压力
```

### 6.5 认知压缩

回答：

```text
能否用更少文字承载更多判断？
能否删除解释，把信息变成可推断？
```

失败风险：

```text
解释欲过强
抽象词堆积
总结腔
信息密度低
```

### 6.6 语言节奏波形

回答：

```text
语言如何压缩、延迟、断裂、释放？
句群如何形成注意力变化？
```

失败风险：

```text
句长均匀
段落均匀
情绪强度均匀
读者匀速滑过
```

### 6.7 现实锚点

回答：

```text
语言是否落到可感知的现实触点？
```

锚点类型：

```text
触感
重量
温度
气味
空间限制
动作代价
生理反应
物体状态
```

失败风险：

```text
只写抽象情绪
没有重量
没有空间
没有物证
描写假
```

### 6.8 局部符号触发

回答：

```text
这段语言是否有可记忆、可复现、可变义的局部符号？
```

失败风险：

```text
读者记不住画面
重要变化没有落点
局部意象不承载状态
象征只是装饰
```

### 6.9 语言预测收益

回答：

```text
读完这一句，读者对语境、人物态度、风险或下一句走向的预测是否改变？
```

失败风险：

```text
语言漂亮但不增加判断
读者没有局部期待
下一句没有压力
```

### 6.10 AI味抑制

回答：

```text
语言是否被平均化、安全化、解释化、模板化？
```

失败风险：

```text
句式平均
抽象词过多
过度总结
解释先于事件
角色声音同质化
缺少语境摩擦
```

### 6.11 语言体验成立

语言体验成立的最低标准：

```text
读者持续获得新信息。
读者能感到语言属于当前语境。
读者不被重复消耗。
读者能从语言中自行推断。
读者能感到节奏变化。
读者能记住局部锚点。
读者读完后多预测一步。
AI味不影响阅读真实感。
```

## 7. 语言单位层级

L2-02 不默认只处理句子。

L2-02 的处理对象统一称为“语言单元”。

语言单元分为六层：

```text
字词层
句子层
句群层
段落层
场景表达层
多章语言层
```

### 7.1 字词层

处理对象：

```text
词语、短语、称谓、动词、形容词、抽象词、术语
```

关注问题：

```text
词是否准确
是否泛化
是否虚
是否有语境来源
是否过度抽象
是否与人物身份不符
```

### 7.2 句子层

处理对象：

```text
单句表达
```

关注问题：

```text
是否有信息增量
是否被语境锁定
是否只是解释
是否存在AI味
是否能删除
是否改变读者判断
```

### 7.3 句群层

处理对象：

```text
连续 2–5 句
```

关注问题：

```text
是否递进
是否重复
是否形成节奏变化
是否有压缩与释放
是否有语义层次
是否每句承担不同功能
```

### 7.4 段落层

处理对象：

```text
一个自然段或功能段
```

关注问题：

```text
是否有段落目标
是否存在语言节奏波形
是否有明确状态变化
是否解释过多
是否能形成段尾推进
是否有局部记忆锚点
```

### 7.5 场景表达层

处理对象：

```text
一个场景内的语言整体
```

关注问题：

```text
人物声音是否区分
场景压力是否进入语言
描写是否有现实锚点
语言是否服务结构推进
是否形成场景记忆点
是否出现平均化表达
```

### 7.6 多章语言层

处理对象：

```text
连续章节或长期文本风格
```

关注问题：

```text
风格是否一致
风格是否僵化
语言是否持续有变化
重要符号是否重复变义
角色声音是否随状态变化
是否出现长期AI味回潮
```

## 8. L2-02 核心任务

L2-02 只做十一件事：

```text
1. 判断单位语言是否存在新增差异
2. 判断语言是否被人物、场景、关系、压力锁定
3. 判断重复是否被压缩或升级
4. 判断解释是否超过必要程度
5. 判断文本是否存在AI味
6. 判断句群是否存在语言节奏波形
7. 判断描写是否具备现实锚点
8. 判断局部符号是否形成记忆触发
9. 判断语言是否改变读者预测
10. 判断输入是否足够支持验收
11. 判断语言问题是否需要回流 L1/L1.5 或建议重路由
```

## 9. 输入边界

### 9.1 允许输入

L2-02 接收来自 L1 / L1.5 的文风语言问题包。

标准输入格式：

```yaml
failure_type: 文风语言失败
failure_signal:
  - 信息增量不足
  - 语境锁定失败
  - 重复
  - 啰嗦
  - 解释欲过强
  - AI味
  - 语言节奏平滑
  - 现实锚点不足
  - 局部符号触发弱
  - 语言预测收益不足
severity: low | medium | high
source_layer: L1 | L1.5
route_target: L2-02
context_scope: 字词 | 句子 | 句群 | 段落 | 场景表达 | 多章语言
```

### 9.2 禁止输入

L2-02 禁止直接接收：

```text
“帮我写得更好看”
“润色一下”
“高级一点”
“更有文采”
“像某某作者”
“直接改正文”
“生成章节”
“套用某一章的文风”
“写成爆款风格”
“模仿某平台热文语气”
```

如需处理正文，必须先由 L1 / L1.5 抽象成文风失败包。

## 11. 主失败仲裁规则

当一个语言单元同时出现多个文风失败时，L2-02 必须先判断主失败。

主失败优先级如下：

```text
P1 信息增量不足
P2 语境锁定失败
P3 重复 / 啰嗦
P4 解释欲过强
P5 AI味
P6 语言节奏平滑
P7 现实锚点不足
P8 局部符号触发弱
P9 语言预测收益不足
P10 验收不可判 / 疑似越界
```

仲裁原则：

```text
先修信息增量，再修重复。
先修语境，再修风格。
先修解释，再修AI味。
先修语言节奏，再修修辞。
先修语言预测收益，再修漂亮表达。
输入不足时，不强行判断。
发现越界时，回 L1.5 重路由。
```

优先级解释：

```text
P1 信息增量不足：
如果一句话没有新增认知，任何修辞都是废话。

P2 语境锁定失败：
如果一句话不属于当前人物、场景、关系和压力，它再顺也会漂。

P3 重复 / 啰嗦：
重复会直接消耗读者时间，降低单位语言收益。

P4 解释欲过强：
解释过多会替读者完成判断，削弱参与感。

P5 AI味：
AI味会破坏个体性、语境感和真实感。

P6 语言节奏平滑：
节奏过平会导致注意力滑落。

P7 现实锚点不足：
缺少现实锚点时，语言没有触感、重量和可信度。

P8 局部符号触发弱：
缺少局部符号触发时，文本难以形成记忆锚点。

P9 语言预测收益不足：
语言预测收益建立在信息、语境、节奏和符号有效的基础上。

P10 验收不可判 / 疑似越界：
输入不足或主问题不属于语言层时，不得硬判。
```

## 12. 文风语言失败类型

### F01 信息增量不足

定义：

```text
单位语言没有提供新的状态、新动作、新信息、新压力、新判断、新感受或新语境变化。
```

表现：

```text
一句话只是换说法。
多句表达同一意思。
删除后不影响读者理解。
读者读完没有认知变化。
语言增加长度，不增加判断。
```

修复规则：

```text
每个有效语言单元必须至少提供一种增量：
状态增量、动作增量、信息增量、压力增量、判断增量、情绪增量、语境增量、符号增量。
```

验收标准：

```text
任意语言单元都能回答：
这句话让读者新知道了什么？
如果删除它，损失什么？
```

---

### F02 语境锁定失败

定义：

```text
语言没有被人物身份、场景压力、关系位置、当前目标和世界规则锁住。
```

表现：

```text
谁都能说这句话。
换个场景也成立。
语言不受权力关系影响。
语言不受危险程度影响。
同一角色不同状态下说话没有变化。
句子像通用模板，不像现场发生。
```

修复规则：

```text
每句话必须被语境约束。
语境包括：人物身份、关系位置、场景压力、当前目标、风险等级、世界规则。
```

验收标准：

```text
如果换人、换场景、换关系后仍不变，则语境锁定失败。
```

---

### F03 重复 / 啰嗦

定义：

```text
信息没有被压缩成新增差异，只是持续重复表达。
```

表现：

```text
同义反复。
情绪反复说明。
状态反复确认。
因果反复解释。
句群只扩大长度，不扩大认知。
多句只在语气上变化，不在信息上变化。
```

修复规则：

```text
重复信息必须合并、删除或升级为新差异。
```

处理方式：

```text
1. 同义重复 → 删除
2. 情绪重复 → 换成行为或物证
3. 状态重复 → 合并为一次明确状态
4. 解释重复 → 转为读者可推断的信息
5. 必要重复 → 递进为强度变化、位置变化或意义变化
```

验收标准：

```text
连续三句不得提供同一层信息。
```

---

### F04 解释欲过强

定义：

```text
文本替读者完成过多理解、判断和情绪归纳。
```

表现：

```text
直接说明人物很害怕。
直接解释规则意义。
直接总结冲突价值。
直接告诉读者该怎么理解。
先解释后发生。
解释超过了因果成立所需。
```

修复规则：

```text
能通过行为、物件、反应、后果让读者推断的内容，不直接解释。
```

验收标准：

```text
解释句必须满足：
没有它，读者会误解关键因果。
否则删除或改为可推断信息。
```

---

### F05 AI味

定义：

```text
语言呈现平均分布、通用表达、解释安全化、个体性不足和语境摩擦不足。
```

表现：

```text
句式过工整。
过度总结。
抽象词过多。
情绪词直接标注。
比喻通用。
角色声音相同。
缺少具体情境摩擦。
像在解释，而不是发生。
语言安全、圆滑、无毛刺。
```

修复规则：

```text
AI味必须通过语境锁定、个体差异、物证化表达、句式破均值来消除。
```

验收标准：

```text
文本必须能回答：
这句话为什么只能出现在这个场景、这个人物、这个时刻？
```

---

### F06 语言节奏平滑

定义：

```text
句群长度、信息密度、情绪强度和注意力波动过于均匀。
```

表现：

```text
每段长度接近。
情绪强度不变。
动作、心理、解释平均分布。
没有短促断点。
没有压迫、延迟、释放。
读者读感匀速滑过。
```

修复规则：

```text
语言单元必须形成语言节奏波形：
压缩、延迟、断裂、释放、停顿、突刺。
```

语言节奏工具：

```text
1. 短句断流
2. 长句压迫
3. 信息延迟
4. 意外提前
5. 留白
6. 反应截断
7. 句群密度变化
8. 句式长度错位
```

验收标准：

```text
一个语言单元内必须存在可感知的节奏变化点。
```

---

### F07 现实锚点不足

定义：

```text
语言没有现实触点，缺少触感、重量、温度、空间、动作代价或可观察物证。
```

表现：

```text
只写抽象情绪。
动作没有重量。
场景没有空间限制。
身体反应模板化。
恐惧只写“发抖”“脸白”。
痛苦只写“撕心裂肺”。
物件状态不参与表达。
```

修复规则：

```text
语言描写必须绑定至少一个现实锚点：
触感、重量、温度、气味、空间限制、动作代价、生理反应、物体状态。
```

验收标准：

```text
读者能否感到：
这个表达有可感知的现实触点，而不是概念说明？
```

---

### F08 局部符号触发弱

定义：

```text
语言单元缺少可压缩、可记忆、可复现、可变义的局部符号节点。
```

表现：

```text
没有核心物件。
没有重复变义的局部符号。
场景发生后没有记忆锚点。
读者记不住画面。
重要变化没有落到可见符号上。
```

修复规则：

```text
重要语言单元应绑定局部符号节点。
局部符号必须能承载状态、关系、冲突、代价或未来压力。
```

验收标准：

```text
读者能否用一个物件、动作、句子或画面记住本语言单元？
```

---

### F09 语言预测收益不足

定义：

```text
单位语言没有改变读者对当前语境、人物态度、局势压力或下一句走向的判断。
```

表现：

```text
读完没有新判断。
下一句没有期待。
信息没有改变当前预测。
语言漂亮但不增加判断。
文本没有形成继续读的局部收益。
```

修复规则：

```text
单位语言必须提供语言层预测收益：
让读者对语境、人物、规则、风险或下一步表达的判断发生变化。
```

验收标准：

```text
读者读完后必须能回答：
我现在比刚才多预测到了什么？
```

---

### F10 验收不可判 / 疑似越界

定义：

```text
输入不足以判断文风语言失败，或主问题不属于文风语言层。
```

典型输入不足：

```text
没有文本片段
没有人物身份
没有场景压力
没有关系位置
没有当前目标
没有失败位置
没有需要验收的语言单元
```

典型越界：

```text
主线发散 → L2-01
人格欲望不足 → L2-03
世界规则空洞 → L2-04
留存付费弱 → L2-05
状态失忆 / 真源污染 → L2-06
```

修复规则：

```text
L2-02 不硬判。
输出“需要补输入”或“建议回 L1.5 重路由”。
```

## 13. AI味检测维度表

L2-02 使用以下维度识别 AI味：

```text
1. 平均句式：
   句子长度、结构、停顿方式过于均匀。

2. 抽象词密度过高：
   大量使用“复杂、深刻、重要、某种、仿佛、似乎”等虚词和泛词。

3. 情绪直标：
   直接写“他很震惊、她很难过、众人很愤怒”。

4. 解释先于事件：
   先告诉读者意义，再让事情发生。

5. 角色声音同质化：
   不同人物说话像同一个旁白。

6. 缺少物证：
   情绪、关系、压力没有落到动作、物件、空间、身体上。

7. 缺少语境摩擦：
   语言不受身份、权力、危险、目标、关系制约。

8. 比喻泛化：
   比喻可套用到任何文本，没有当前场景来源。

9. 总结腔：
   频繁使用结论式、概括式、教学式表达。

10. 安全化表达：
   语言过度圆滑，避开尖锐差异和不舒服的真实细节。
```

AI味判定原则：

```text
单项出现不等于 AI味。
三项以上同时出现，且影响语境真实感时，判为 AI味风险。
```

## 14. 核心规则

### R01 单位语言必须有增量

```text
每个有效语言单元必须提供新增差异。
没有新增差异，就删除、合并或升级。
```

### R02 语言必须语境锁定

```text
语言必须属于当前人物、当前场景、当前压力和当前关系。
```

### R03 信息必须压缩

```text
多句能表达的内容，优先压成更高密度的一句。
但不得牺牲必要节奏和可读性。
```

### R04 解释必须最小化

```text
能让读者推断的，不直接解释。
必须解释的，只解释关键因果。
```

### R05 抽象情绪必须物证化

```text
少写“他很害怕”。
多写让读者判断他害怕的行为、反应、物件状态和身体变化。
```

### R06 语言节奏必须有波形

```text
文本不能匀速前进。
必须有压缩、延迟、断点、释放和变化。
```

### R07 局部符号必须服务表达

```text
局部符号不是装饰。
局部符号必须承载状态、关系、冲突、代价或未来压力。
```

### R08 语言必须带来预测收益

```text
语言不是为了好看。
语言必须改变读者对语境、人物、风险、规则或下一步表达的预测。
```

### R09 文风层不得越界

```text
L2-02 只负责“怎么说出来”。
不负责“写什么”，不负责“怎么发生”，不负责“读者是否付费”。
```

### R10 闭环必须回流

```text
L2-02 修复作业完成后，必须回流 L1 / L1.5 验收。
不得直接进入正文生产。
```

## 15. 修复动作库

L2-02 可输出以下修复动作：

```text
信息增量标注：
逐句标注每句话提供的新信息类型。

语境锁定：
将句子绑定人物身份、关系位置、场景压力与当前目标。

重复压缩：
删除同义重复，合并低密度句群。

解释削减：
删除不必要解释，只保留关键因果说明。

解释转物证：
将抽象解释改为行为、物件、反应、后果。

句式破均值：
打破整齐、安全、平均的句式结构。

语言节奏波形重排：
重组句群长度、停顿、延迟、释放与突刺。

现实锚点补强：
补入触感、重量、温度、气味、空间限制或动作代价。

局部符号绑定：
建立可记忆、可复现、可变义的局部符号。

语言预测收益强化：
让句子改变读者对语境、人物态度、风险或下一句走向的判断。

建议回 L1.5 重路由：
当主失败不是文风语言问题时，生成疑似越界说明、建议目标和回 L1.5 重路由要求。
```
