# L2 输入与证据上下文边界

> 状态：冻结（L2-BND-V2）  
> 层级：L2 正式能力层 · 输入边界唯一真源  
> 生产运行时：v1（`xcue.failure-packet/1.0` + `xcue.l15-route-report/1.0`）

本文件是 L2 **业务主输入**、**证据上下文地位**、**六模块上下文权限**、**重路由与输入不足动作** 的唯一权威说明。  
L2-00、L2-99 与各模块定义只引用本文件，不再重复维护多套表述。

---

## 1. L2 业务主输入

L2 唯一合法业务主输入是：

```text
L1 失败包 + L1.5 路由决策
```

当前生产运行仍为 v1，实际对象：

- 失败包：`xcue.failure-packet/1.0`
- L1.5 报告：`xcue.l15-route-report/1.0`，且 `final_status = ROUTED` 时含有效 `target_module`

L2 **不得**仅凭以下任一组合启动正式能力：

```text
章节正文
IR 文件
前序章节
用户一句“帮我修”
```

章节正文**单独**进入 L2 时必须拒绝：

```text
L2_UPSTREAM_PACKAGE_REQUIRED
```

---

## 2. 正文、IR 与前序章节的地位

统一定义为：

```text
证据上下文 + 修复目标上下文
```

它们**不是**：

```text
问题存在性的权威来源
模块路由的权威来源
新的 L1 输入替代品
```

允许 L2 读取正文，是为了：

1. 验证失败包证据；
2. 在已路由模块内细化根因；
3. 确定修改位置；
4. 制定修复规则；
5. 制定可执行动作；
6. 设计验收条件；
7. 判断输入是否不足或路由是否明显不成立。

**不得**利用全文重新进行全局问题扫描。

---

## 3. L2 允许的判断

L2 允许：

1. 验证上游失败证据是否存在；
2. 判断当前上下文是否足够；
3. 在目标模块内部确定子问题；
4. 细化模块内部根因；
5. 排列模块内部主问题和次级问题；
6. 输出修复规则；
7. 输出修复动作；
8. 输出禁止修改范围；
9. 输出必须保留内容；
10. 输出验收条件；
11. 请求返回 L1.5 重新路由。

L2 不允许：

1. 重新决定全章是否存在问题；
2. 重新选择目标 L2 模块；
3. 直接将任务改派到另一个模块；
4. 发现和处理与当前路由无关的其他问题；
5. 直接改写正式正文；
6. 生成候选正文；
7. 进行发布裁决；
8. 判断平台运营效果；
9. 修改 L1 失败包；
10. 修改 L1.5 路由结果。

---

## 4. 路由不成立时

L2 发现当前目标模块明显不适合时，只能输出：

```text
RETURN_TO_L1_5
```

并提供：

- 请求重路由原因；
- 证据不足点；
- 建议问题域。

**不得**直接写入新的 `target_module`。

---

## 5. 上下文不足时

缺少完成修复设计所必需的正文、IR、前章或状态材料时，输出：

```text
INPUT_REQUIRED
```

并明确缺失材料。

**不得**用模型猜测补齐缺失事实。

## 6. 六模块上下文权限矩阵

### L2-01 叙事结构

| 上下文 | 权限 |
|---|---|
| 当前章节完整正文 | 允许 |
| failure evidence | 允许 |
| 章节顺序 | 允许 |
| 与连续性问题直接相关的相邻章节片段 | 允许（非全历史） |
| 全部历史章节 | 默认禁止 |
| IR / runtime | 禁止 |

用途：因果、主线、顺序、场景切换、结构闭合。

### L2-02 文风语言

| 上下文 | 权限 |
|---|---|
| 当前章节完整正文 | 允许 |
| failure evidence | 允许 |
| 明确的项目文风约束 | 允许 |
| 角色 IR / 世界 IR | 仅作不可改变约束，禁止重新设计 |
| 前序章节 / runtime | 禁止 |

用途：句式、信息增量、解释腔、重复、语气、表达密度。

### L2-03 角色心理

| 上下文 | 权限 |
|---|---|
| 当前章节完整正文 | 允许 |
| failure evidence | 允许 |
| 涉及角色的角色 IR | 允许（仅相关角色） |
| 与该角色心理连续性有关的前序片段 | 允许 |
| 当前角色状态 | 允许 |
| 无关角色全部资料 | 禁止 |

用途：欲望、目标、刺激、行为、选择、结果、心理变化和关系压力。

### L2-04 创意设定

| 上下文 | 权限 |
|---|---|
| 当前章节完整正文 | 允许 |
| failure evidence | 允许 |
| 世界与设定 IR | 允许（本章使用到的设定） |
| 项目规则 | 允许 |
| 当前章节使用到的设定状态 | 允许 |
| 前后事实硬冲突宣判 | 禁止（属 L2-06） |

用途：规则、限制、代价、选择压力、差异性、持续玩法。

### L2-05 市场体验

| 上下文 | 权限 |
|---|---|
| 当前章节完整正文 | 允许 |
| failure evidence | 允许 |
| 明确的目标读者与内容侧阅读目标 | 允许 |
| 当前章节阅读窗口 | 允许 |
| 投流 / 封面点击率 / 真实收入 / 平台推荐 / 广告 / 外部运营数据 | 默认禁止 |

用途：入口、即时收益、继续阅读动力、认知负担、弃读风险、章末追读。

### L2-06 系统一致性

| 上下文 | 权限 |
|---|---|
| 当前章节完整正文 | 允许 |
| failure evidence | 允许 |
| 相关前序章节 | 允许 |
| IR | 允许 |
| 项目规则 | 允许 |
| runtime 状态 | 允许 |
| 已确认故事事实 | 允许 |
| 文件路径 / API / Schema / Git / 治理 / 运行环境问题 | 禁止 |

用途：人物状态、世界规则、关系、知情范围、时间、位置和事实继承。

---

## 7. 与 L3 的执行边界

- L2 输出修复单与验收条件，**不**写候选正文。
- 候选正文生成属于 L3；L2 阶段不得创建 `chapters/_candidates/` 产物。
- L2 修复单中的 `重路由请求.禁止直接指定新目标模块` 必须为 `true`。

---

## 8. 旧兼容入口

`--failure-packet` 直连入口标记为 `LEGACY_EXPLICIT_ONLY`：

- 不得作为正式流水线入口；
- 不得允许章节正文单独调用；
- 不得自动补造 L1.5 路由报告。

正式流水线：`failure packet → L1.5 → L2（--l15-report）→ L3`。
