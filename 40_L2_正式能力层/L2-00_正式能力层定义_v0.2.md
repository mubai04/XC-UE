## 1. L2 定位

L2 是 XC-UE 的专题能力层。

它不是章节生产层。
它不是正文生成层。
它不是测试层。
它不是轻验收记录。
它不是临时补丁。
它不是图片展示层。
它不是 Cursor 执行协议。
它不是旧对话资料库。
它不是母法资产库。

L2 的职责是：

```text
接收 L1 / L1.5 传来的问题类型、失败原因、路由结果，
将其转译为可执行的专题能力规则、修复动作与回流验收标准。
```

一句话：

```text
L2 不判断哪里坏。
L2 定义怎么修。
```

---

## 2. L2 在系统中的位置

```text
L0：总工程图 / 系统边界 / 总导航
↓
L1：闸门层 / 判断问题是否存在
↓
L1.5：路由矩阵层 / 判断问题应派发到哪里
↓
L2：专题能力层 / 定义怎么修、怎么生成能力、怎么验收能力
↓
L3：执行协议层 / 具体正文、章节、项目产出、Cursor 操作
```

职责分工：

```text
L1 判断“哪里坏”。
L1.5 判断“该交给谁修”。
L2 定义“怎么修、修到什么程度、修完回哪里验收”。
L3 执行“具体怎么做”。
```

L2 不得越过 L1.5 直接派单。

L2 发现越界问题时，只能输出：

```text
疑似越界
建议目标能力
回 L1.5 重路由
```

## 3. L2 上游真源规则

L2 只能从以下来源接收信息：

```text
P0：当前正式 Markdown 真源
P1：已人工确认的候选 Markdown 规则
P2：L0 / L1 / L1.5 已确认工程文件
P3：人工转译后的外部资料规则
P4：测试记录与失败断点
P5：派生说明材料
P6：临时实验文件
P7：废弃章节与旧系统文件
```

规则：

```text
P6 / P7 不得进入正式 L2 定义。
P5 只能做说明，不得新增规则。
P3 必须先转译，不能原文塞入 L2。
P1 只能作为候选规则，未经确认不得覆盖 P0。
P0 优先级最高。
```

当图片与 Markdown 冲突时：

```text
以 Markdown 为准。
```

当对话建议与 Markdown 冲突时：

```text
以 Markdown 为准。
```

当旧对话记忆与当前上传文件冲突时：

```text
以当前上传文件为准。
```

## 4. L2 输入

> **冻结真源**：`L2_输入与证据上下文边界.md`（L2-BND-V2）。本节只保留摘要。

L2 唯一合法业务主输入：

```text
L1 失败包 + L1.5 路由决策
```

正文、IR、前序章节仅为 **证据上下文 + 修复目标上下文**，不是路由权威。

章节正文单独进入 L2 → `L2_UPSTREAM_PACKAGE_REQUIRED`。

L2 禁止：全局重诊断、直接改派模块、写候选正文、修改 L1/L1.5 结果。

路由不成立 → `RETURN_TO_L1_5`；上下文不足 → `INPUT_REQUIRED`。

L2 允许接收（经 L1/L1.5 转译后）：

```text
L0 总工程图
L1 闸门失败类型
L1.5 路由结果
已确认的能力边界
已确认的项目真源图
外部资料经人工转译后的根因链
经人工摘录后的问题总览规则
经人工摘录后的 AI 写作缺陷规则
已验证的失败断点
```

L2 禁止直接作为主输入：

```text
章节正文草稿（单独）
第01章、第02章、第03章实验结果
临时章节状态卡
轻验收记录
旧章节生产 SOP
单一作品设定
临时故事项目需求
旧对话原文
母法资料全文
未转译的论文/文章/截图
```

如果必须使用外部资料，必须经过转译：

```text
原始资料
↓
问题根因
↓
能力模块
↓
失败触发条件
↓
修复动作
↓
回流验收标准
↓
L2 可用规则
```

说明：

```text

```

### 7.1 L2-01 叙事结构能力

回答：

```text

```

负责：

```text

主线约束
顺序控制
结构节奏压缩
心理映射
路径剪枝
```

解决：

```text

时间线混乱
主线发散
因果链断裂
多路径未剪枝
结构筛选失败
信息顺序失控
情节平滑
心理映射失败
```

不负责：

```text
句子好不好
人物人格深不深
世界设定新不新
读者是否付费
长期状态是否继承
```

## 8. L2 与 L1 / L1.5 的接口

L1 输出失败类型。

示例：

```yaml
source_gate: L1-01
failure_type: 叙事结构失败
failure_signal:
  - 主行动线不清
  - 多路径并存
severity: medium
```

L1.5 输出路由结果。

示例：

```yaml
source_layer: L1.5
primary_failure: 主线发散
route_target: L2-01
repair_direction:
  - 主线锁定
  - 路径剪枝
return_to:
  - L1-01
```

L2 接收后输出能力作业。

示例：

```yaml
ability: L2-01_叙事结构能力
primary_failure: 主线发散
repair_rule: 一个结构单元只能保留一条主行动线
repair_action:
  - 列出所有可推进方向
  - 选择唯一主路径
  - 将其余路径降级为压力、伏笔、误导或删除
acceptance:
  - 主行动线唯一
  - 多余路径已剪枝
  - 结构末端留下下一步问题
return_to:
  - L1-01
  - L1.5
```

## 11. L2 生成与修正规程

任何 L2 文件生成或修正，必须按以下顺序：

```text
1. 确认上游真源
2. 确认所属 L2 模块
3. 确认能力边界
4. 确认不负责什么
5. 定义输入格式
6. 定义输出格式
7. 定义主失败仲裁
8. 定义失败类型
9. 定义修复动作
10. 定义回流验收
11. 定义禁止项
12. 标记当前状态
```

禁止直接进入：

```text
写新规则
画新图
生成正文
继续扩写
```

## 12. L2 禁止项

L2 禁止：

```text
从章节实验倒推正式架构
把某一章的生产经验当通用能力
把轻验收写成 L2
把正文生成流程写成 L2
把项目设定写成 L2
把临时补丁命名为正式能力
把解释层材料原样塞进能力层
把母法资料全文复制进 L2
未经转译直接使用旧对话内容
让章节正文污染能力定义
在 XC-UE 活跃工程目录内建立母法资产文件夹
让 Cursor / AI 自动索引旧对话原文或母法原文
让图片覆盖 Markdown
让 L2 绕过 L1.5 直接派单
让 L2 直接变成 L3 执行协议
让 L2 直接生成正文
```
