# MANIFEST 文件权限与真源表 v0.2.1

> 状态：Project Harness 本地权限真源  
> 用途：规定 TP-001 内哪些文件是正式输入，哪些只读，哪些可写，哪些禁止作为真源。  
> 适用范围：本测试项目壳，不覆盖 XC-UE 系统层。

---

## 1. 真源优先级

本项目内部真源优先级：

```text
IR/
↓
chapters/chxx.md 正式正文事实
↓
tests/ 测试记录
↓
logs/ 执行与断点记录
↓
runtime/ 状态变化记录
↓
_legacy_root_inputs/ 旧输入参考
```

解释：

- IR 是当前项目正式输入。
- 正文事实若与 IR 冲突，必须触发 IR 同步或 L2-06 状态校验。
- runtime 记录状态变化，不自动推演，不升 L4。
- _legacy_root_inputs 只作参考，不作正式输入。

---

## 2. 文件权限矩阵

| 路径 | 作用 | 默认权限 | 是否真源 | 说明 |
|---|---|---:|---:|---|
| `00_项目说明.md` | 项目壳说明 | 读 / 条件写 | 是 | 只定义测试壳边界 |
| `MANIFEST_文件权限与真源表.md` | 本地权限表 | 读 / 条件写 | 是 | Cursor 执行前必须读取 |
| `IR/` | 当前项目正式输入 | 读 / 条件写 | 是 | L1-L3 默认读取来源 |
| `IR/IR-99_输入完整性检查.md` | 输入准入检查 | 读 / 条件写 | 是 | 决定是否允许跑 L1 |
| `runtime/` | 状态变化记录 | 读 / 追加写 | 否 | 不自动推演，不升 L4 |
| `runtime/RT-00_运行时索引.md` | runtime 回写索引 | 读 / 条件写 | 是 | 规定 runtime 如何回写 IR |
| `chapters/chxx.md` | 正式正文 | 读 / 默认不写 | 是 | 候选通过后人工合并 |
| `chapters/_candidates/` | 候选正文 | 读 / 写 | 否 | L3 默认输出区 |
| `tests/` | 测试记录 | 读 / 追加写 | 是 | 不覆盖旧记录 |
| `tests/L3_执行任务单.md` | L3 执行入口 | 读 / 条件写 | 是 | 承接 L3 v0.1.2 |
| `logs/` | 日志记录 | 读 / 追加写 | 是 | 执行、断点、修复、复验 |
| `snapshots/` | 快照 | 读 / 写 | 否 | 测试前后保存 |
| `_legacy_root_inputs/` | 旧输入兼容 | 只读 | 否 | 不进入默认执行链路 |

---

## 3. 默认读取规则

L1 / L1.5 / L2 / L3 默认读取：

```text
IR/
chapters/
tests/
logs/
runtime/
MANIFEST_文件权限与真源表.md
```

默认不读取：

```text
_legacy_root_inputs/
```

除非任务单明确写：

```yaml
允许读取旧输入: 是
用途: 迁移 / 对照 / 人工复核
```

---

## 4. 阻断条件

出现以下情况必须阻断：

1. 没有 IR-00 项目索引。
2. 没有 MANIFEST 文件权限表。
3. L3 任务要求直接覆盖正式正文。
4. Cursor 试图读取 `_legacy_root_inputs/` 作为正式输入。
5. runtime 试图自动推演剧情。
6. 候选正文试图自动合并正式正文。
7. IR 与正文冲突但未记录状态。
8. 测试记录被覆盖而非追加。
