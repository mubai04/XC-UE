# L3-05 日志记录协议 v0.1.2

> 状态：候选真源 / 未定版 / 待测试  
> 定位：规定 L3 执行日志、断点记录、修复记录、diff 摘要如何写。  

---

## 1. 日志类型

日志分为：

1. 执行日志
2. 断点记录
3. 修复记录
4. 回滚记录
5. 图文一致性记录
6. 复验记录
7. diff / 变更摘要

---

## 2. 执行日志格式

```yaml
执行日志:
  执行编号: L3RUN-0001
  时间: ""
  执行人: Cursor / 人工 / ChatGPT辅助
  来源任务: ""
  读取文件: []
  修改文件: []
  新增文件: []
  候选产物: []
  备份文件: []
  diff摘要文件: ""
  输出产物: []
  执行结果: 已完成 / 失败 / 阻断 / 需补输入 / 待复验
  下一步: ""
```

---

变更摘要:

  候选产物: []

    - ""

    - ""

---

## 4. 断点记录格式

```yaml
断点记录:
  断点编号: BP-0001
  发生位置: ""
  输入材料: ""
  系统卡住点: ""
  预期行为: ""
  实际行为: ""
  归属候选: L1 / L1.5 / L2 / L3 / 外部运营层
  是否需要修系统: 是 / 否
```

---

修复记录:

  修复文件: []
  回流验收: ""
  结果: ""
```

  复验闸门: L1-01 / L1-02 / L1-03 / L1.5
  复验结果: 通过 / 失败 / 不可判
  证据: ""
  后续动作: 任务关闭 / 二次修复 / 断点记录 / 人工复核
```


## 6. 禁止

禁止：

- 把聊天感受当断点
- 无编号记录
- 不写来源
- 不写回流位置
- 修了系统但不写修复记录
- 执行后不写 diff / 变更摘要
- 有候选正文但不写候选产物路径
