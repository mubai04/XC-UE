# L3-06 IR 输入映射协议 v0.1.2

> 状态：候选真源 / 未定版 / 待测试  
> 定位：规定 Project Harness 内 IR 文件如何映射为 L3 可执行输入。  

---

## 1. IR 定位

IR 是项目中间表示层。

IR 不等于 XC-UE 系统真源。  
IR 不等于正文。  
IR 不等于 L2 能力定义。  

IR 负责把项目材料标准化为 XC-UE 可读输入。

---

## 3. IR 到 L1 / L2 的映射

| IR 文件 | 主要供给 | 用途 |
|---|---|---|
| IR-00 项目索引 | L1 / L1.5 / L3 | 确认项目对象 |
| IR-01 立项卡 | L1-01 / L1-02 / L2-05 | 项目卖点与目标平台 |
| IR-02 世界约束 | L1-01 / L2-04 / L2-06 | 世界规则与约束 |
| IR-03 角色动机表 | L1-01 / L2-03 | 人格驱动与心理迁移 |
| IR-04 事件链 | L1-01 / L2-01 / L2-06 | 因果推进与状态继承 |
| IR-05 章节目标表 | L1-01 / L1-03 / L2-01 | 章节结构与章末钩子 |
| IR-06 读者预期表 | L1-02 / L2-05 | 投入、追读、传播理由 |
| IR-07 发布状态表 | L1-03 / L2-05 | 发布锁验收 |
| IR-08 状态快照 | L2-06 / L3 | 状态继承与回滚 |

## 4. IR 优先级

当 IR 与正文冲突：

- 正文事实优先于未更新 IR。
- 但必须触发 L2-06 状态校验。
- 修复后必须同步 IR 或标记 IR 过期。

当 IR 与 L2 能力文件冲突：

- L2 能力文件优先。
- IR 只能作为项目输入，不得改写 L2 定义。

当 IR 与图片冲突：

- Markdown 真源优先。
- 图片不得覆盖 IR。

---

## 5. L3 执行前 IR 校验

执行正文任务前必须检查：

1. 是否存在 IR-00 项目索引。
2. 是否存在至少一个章节目标表。
3. 是否存在角色动机输入。
4. 是否存在世界约束输入。
5. 是否存在读者预期输入。
6. 是否存在状态快照或明确说明“初始状态”。

缺失时，L3 任务状态为：

```text
需补输入
```

---

## 5.1 IR 状态字段

每个 IR 文件在被 L3 读取前，必须具备或被补记以下状态：

```yaml
IR状态:
  状态: 当前 / 过期 / 待同步 / 冲突
  最近同步来源: chapters / tests / logs / runtime / 人工
  冲突对象: ""
  是否允许用于执行: 是 / 否
  需要同步到: ""
```

判定规则：

1. 当前：可作为 L3 执行输入。
2. 过期：不得直接作为正文候选生成输入，需先同步。
3. 待同步：可以作为参考，但必须标注风险。
4. 冲突：必须回 L2-06 状态校验或人工复核。
