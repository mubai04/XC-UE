# L3-04 版本与回滚协议 v0.1.2

> 状态：候选真源 / 未定版 / 待测试  
> 定位：规定版本、备份、候选产物、正式合并、回滚流程。  

---

## 1. 基本原则

1. 修改前必须备份。
2. 关键文件必须保留版本号。
3. 正式文件与候选文件分开。
4. 回滚必须记录原因。
5. 不允许无痕覆盖。
6. 候选章节不得自动合并到正式章节。

---

## 2. 版本命名

候选文件：

```text
文件名_candidate_v01.md
```

修正版：

```text
文件名_v0.1.2_自检修正版.md
```

备份：

```text
文件名.bak_YYYYMMDD_HHMM
```

快照：

```text
snapshot_YYYYMMDD_HHMM/
```

diff 摘要：

```text
DIFF_文件名_YYYYMMDD_HHMM.md
```

---

## 3. 候选正文合并规则

候选正文合并到正式正文必须满足：

1. 候选正文已生成。
2. diff / 变更摘要已生成。
3. L1 / L1.5 复验通过。
4. 用户或人工操盘者明确确认合并。
5. 正式正文已备份。
6. 合并记录写入 logs / tests。

未满足以上条件，候选正文不得覆盖正式正文。

---

## 4. 回滚触发条件

触发回滚：

- 修改结果破坏真源
- 文件路径错误
- 覆盖错误文件
- L1 复验失败且修复方向错误
- 用户要求回退
- 图文一致性审计失败
- 候选合并后破坏状态继承

---

```yaml
  是否已恢复: 是 / 否
  后续复验位置: ""
```

---

## 6. 禁止

禁止：

- 无备份回滚
- 删除错误版本
- 只凭对话记忆回滚
- 用归档文件覆盖当前真源
- 候选正文自动覆盖正式正文
