# L3-02 正文生成与改写任务协议 v0.1.2

> 状态：候选真源 / 未定版 / 待测试  
> 定位：规定候选正文生成、扩写、改写、局部修复任务如何执行。  

---

## 1. 基本边界

L3-02 可以执行正文相关任务，但不判断正文好坏。

正文好坏由 L1 / L2 判断。

L3-02 只回答：

- 生成哪个候选章节
- 改哪个候选段落
- 根据哪个修复单改
- 读取哪些 IR 输入
- 输出到哪个候选文件
- 原文如何备份
- 修完回哪里验收

---

## 2. 权限降级原则

v0.1 中的“新章生成”统一降级为：

```text
候选正文生成
```

v0.1 中的“章节扩写 / 局部改写”统一降级为：

```text
候选正文扩写 / 候选正文改写 / 指定范围局部修复
```

默认不得直接覆盖：

```text
chapters/ch01.md
chapters/ch02.md
```

默认输出：

```text
chapters/_candidates/
```

---

## 3. 正文任务类型

允许任务：

1. 候选正文生成
2. 候选章节扩写
3. 候选局部改写
4. 删除冗余候选版
5. 补章末钩子候选版
6. 补状态继承候选版
7. 根据 L2 修复单重写指定段落候选版
8. 生成多个候选版本

禁止任务：

1. 无任务单自由写文。
2. 无 IR 输入自由写文。
3. 无备份覆盖原文。
4. 改动未指定章节。
5. 用当前灵感覆盖 L2 修复方向。
6. 让正文改写反向修改系统规则。
7. L3 执行后直接标记正文通过。

```yaml
  来源修复单: ""
  IR输入文件: []
  目标章节: chapters/ch01.md
  修改范围: 全章 / 指定段落 / 指定行 / 指定场景
  必须保留: []
  必须删除: []
  必须新增: []
  禁止项: []
  字数范围: ""
  输出方式: 另存候选 / 追加候选
  输出文件: chapters/_candidates/ch01_candidate_v01.md
  是否允许覆盖正式正文: 否
  回流验收位置: L1-01 / L1-02 / L1-03 / L1.5
```

---

## 5. 候选版本规则

默认输出：

```text
chapters/_candidates/ch01_candidate_v01.md
chapters/_candidates/ch01_candidate_v02.md
```

候选文件头部必须写：

```text
来源修复单：
读取 IR：
修改范围：
禁止项：
回流验收位置：
```

---

## 6. 回流规则

正文执行完成后必须回流：

- 结构类修复 → L1-01
- 文风类修复 → L1-01
- 角色类修复 → L1-01
- 设定类修复 → L1-01
- 市场体验类修复 → L1-02 / L1-03
- 系统一致性修复 → L1-00 / L1-01 / L1.5

L3 执行成功只代表操作完成，不代表 L1 / L2 判断通过。
