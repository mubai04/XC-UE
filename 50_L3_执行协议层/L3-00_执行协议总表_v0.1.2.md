## 0. L3 定位

L3 不是新的创作理论层。

L3 是执行协议层，负责把上游判断变成标准执行任务：

L1 判断结果  
→ L1.5 路由结果  
→ L2 修复方向  
→ L3 执行任务  
→ 文件落盘  
→ 日志记录  
→ 回 L1 / L1.5 验收

一句话：

> L3 = 判断结果到本地执行的操作协议层。

---

## 1. L3 负责什么

L3 负责：

1. 接收 L1 / L1.5 / L2 / L2-99 的执行输入。
2. 转译为 Cursor 或人工可执行任务。
3. 指定读取文件、修改文件、输出文件、日志文件。
4. 规定正文生成、扩写、改写、局部修复的执行格式。
5. 规定版本、备份、回滚、禁止覆盖。
6. 规定修复后如何回填验收记录。
7. 规定执行失败如何记录断点。
8. 规定 IR 输入如何映射到执行任务。
9. 规定 Project Harness 如何作为测试项目壳运行。

---

## 2. L3 不负责什么

L3 不负责：

1. 判断作品好不好。
2. 判断读者是否投入。
3. 判断发布锁是否通过。
4. 判断主失败归属。
5. 替 L1.5 路由。
6. 替 L2 做能力判断。
7. 直接覆盖 Markdown 真源。
8. 无任务单直接改正文。
9. 无备份直接覆盖原文件。
10. 生成新系统层。
11. 把测试项目壳当 XC-UE 系统真源。
12. 把执行成功等同于 L1 / L2 判断通过。

---

## 3. L3 执行状态机

所有 L3 任务必须落入以下状态机：

```text
待执行
↓
输入校验通过
↓
目标文件确认
↓
权限校验通过
↓
备份完成
↓
执行中
↓
候选产物已生成
↓
diff / 变更摘要已生成
↓
日志已记录
↓
回填完成
↓
待 L1 / L1.5 复验
↓
复验结果回写
↓
任务关闭 / 进入二次修复 / 进入断点记录
```

异常状态：

```text
需补输入
阻断
执行失败
回滚待执行
回滚完成
人工复核
```

L3 任务不得跳过：

- 输入校验
- 权限校验
- 备份
- 日志
- 回流验收位置

## 4. 文件权限矩阵

| 区域 / 文件类型 | 默认权限 | 说明 |
|---|---:|---|
| L0-L2 / L1.5 真源文件 | 只读 | 不得由 L3 默认修改 |
| L3 协议文件 | 可读 / 条件写 | 仅版本升级任务可改 |
| Project Harness 项目壳 | 可读 / 可写 | 当前测试项目主要执行区 |
| IR 目录 | 可读 / 条件写 | 只根据 IR 任务单填充 |
| runtime 目录 | 可读 / 可写 | 记录价值函数演化与状态迁移 |
| chapters/chxx.md 正式正文 | 默认只读 | 除非人工确认合并 |
| chapters/_candidates/ | 可写 | 默认输出候选正文 |
| tests/ | 追加写 | 测试记录不得覆盖旧记录 |
| logs/ | 追加写 | 断点、修复、执行日志 |
| snapshots/ | 可写 | 保存测试前后快照 |
| 废弃实验归档 | 只读 | 不得反向覆盖当前工程 |

## 7. 变更摘要要求

每次 L3 执行后必须输出变更摘要：

```yaml
变更摘要:
  改了什么: ""
  为什么改: ""
  依据哪个修复单: ""
  新增了哪些文件: []
  修改了哪些文件: []
  覆盖了哪些文件: []
  备份在哪里: []
  候选产物在哪里: []
  待复验位置: ""
```

如果无法生成变更摘要，则 L3 执行状态不得标记为“已完成”。

---

## 8. L3 执行顺序

1. 校验任务来源。
2. 校验 IR / Project Harness 输入。
3. 校验目标文件。
4. 校验是否允许改正式正文。
5. 校验权限矩阵。
6. 备份目标文件。
7. 执行文件读取 / 修改 / 生成。
8. 输出候选产物。
9. 生成 diff / 变更摘要。
10. 记录执行日志。
11. 回填验收位置。
12. 标记待复验闸门。
13. 若失败，写入断点记录。

---

## 9. L3 最小验收

L3 通过标准：

1. 能明确接收上游任务。
2. 能明确知道读哪些文件。
3. 能明确知道改哪些文件。
4. 能明确知道禁止改哪些文件。
5. 能明确知道输出到哪里。
6. 能明确知道如何备份。
7. 能明确知道如何回滚。
8. 能明确知道回哪里验收。
9. 能明确记录日志。
10. 能明确生成 diff / 变更摘要。
11. 能明确记录复验结果。
12. 能在复验失败后进入二次修复或断点记录。
13. 不越权判断内容质量。
14. 执行成功不等于 L1 / L2 复验通过。
