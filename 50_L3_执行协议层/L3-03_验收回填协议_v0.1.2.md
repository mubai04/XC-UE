# L3-03 验收回填协议 v0.1.2

> 状态：候选真源 / 未定版 / 待测试  
> 定位：规定修复产物如何回填到 L1 / L1.5 / tests / logs。  

---

## 1. 回填原则

1. 修复完成不等于通过。
2. L3 执行成功只代表操作完成，不代表 L1 / L2 判断通过。
3. 必须回原闸门复验。
4. L2 修复产物必须有回流位置。
5. L3 只负责写回记录，不负责判通过。
6. 回填记录不得覆盖原始失败记录。
7. 每个 L3 输出必须绑定待复验闸门、复验文件、复验问题。

---

| 修复来源 | 回填位置 |
|---|---|

---

## 4. 禁止

禁止：

- 修完直接标通过
- 跳过 L1 复验
- 删除旧失败记录
- 用新结果覆盖断点原因
- 把 L3 执行成功等同于 L1 通过
- 无复验闸门就结束任务
---

## 5. 复验结果回写

L3 任务送回 L1 / L1.5 后，必须记录复验结果：

```yaml
复验结果回写:
  来源执行: L3RUN-0001
  复验闸门: L1-01 / L1-02 / L1-03 / L1.5
  复验文件: ""
  复验问题: ""
  复验结果: 通过 / 失败 / 不可判
  后续状态: 任务关闭 / 二次修复 / 断点记录 / 人工复核
  回写位置: ""
```

规则：

1. 复验通过 → 任务关闭。
2. 复验失败 → 进入二次修复或 L1.5 重路由。
3. 复验不可判 → 写入断点记录。
4. 任何复验结果不得覆盖原失败记录。
