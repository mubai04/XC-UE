## 冻结前三项确认

### 1. 终态优先级（已冻结）

```text
若存在 HARD_GUARD
  → status = SCREENING_REJECT
  → 若同时存在 AUDIT_BLOCKER
      → semantic_audit_status = AUDIT_BLOCKED
      → audit_reason_type 保留 API/证据/输入原因

若无 HARD_GUARD，且 API/证据/输入结构不可用
  → status = AUDIT_BLOCKED
  → semantic_audit_status = AUDIT_BLOCKED
```

实现位置：[`L1决策角色.py`](../00_工程总控/工程执行层/L1工程/L1决策角色.py) `聚合终态()`

### 2. 失败包与审计阻断项分离（已冻结）

| decision_role | failure_packet | audit_blockers | 路由 L1.5/L2 |
|---|---|---|---|
| DIAGNOSTIC | 否 | 否 | 否 |
| HARD_GUARD | 是 | 否 | 是（按 gate_rules） |
| CONTENT_DECISION | 是 | 否 | 是（按 gate_rules） |
| AUDIT_BLOCKER | 否 | 是 | 否 |

产物：

- `{run_id}_failure_packet.json`：仅 HARD_GUARD + CONTENT_DECISION
- `{run_id}_audit_blockers.json`：仅 AUDIT_BLOCKER
- `{run_id}.json`：含 `semantic_audit_status`、`audit_blockers` 字段

实现位置：[`失败包生成.py`](../00_工程总控/工程执行层/L1工程/失败包生成.py) `分拆阻断项()`

### 3. 组合态测试矩阵（已覆盖）

| 硬护栏 | 语义状态 | 最终 status | semantic_audit_status |
|---|---|---|---|
| 失败 | PASS | SCREENING_REJECT | PASS |
| 失败 | BLOCKED | SCREENING_REJECT | AUDIT_BLOCKED |
| 通过 | BLOCKED | AUDIT_BLOCKED | AUDIT_BLOCKED |
| 通过 | FAIL | SCREENING_REJECT | FAIL |
| 通过 | REVIEW | REVIEW_REQUIRED | REVIEW |
