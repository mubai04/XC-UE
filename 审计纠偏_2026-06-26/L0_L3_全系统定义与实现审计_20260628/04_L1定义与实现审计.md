## 2. 失败包应包含什么

**Schema：** `公共组件/结构定义/失败包结构.json`

必填顶层：`schema_version`, `pipeline_run_id`, `stage_run_id`, `status`, `failure_count`, `items`

每项必填：`闸门`, `名称`, `状态`, `说明`, `证据`, `严重级别`, `失败类型`, `候选模块`, `回流验收位置`, `修复方向`, `decision_role`, `blocking`

**代码过滤规则（`失败包生成.py`）：** 仅 `blocking=True` 且 `decision_role ∈ {HARD_GUARD, CONTENT_DECISION}` 进入失败包；**DIAGNOSTIC 项排除**。

## 4. L1-01 / L1-02 / L1-03 先后关系

**定义默认顺序：** L1-00 → L1-01 → L1-02 → L1-03（L0 L113–120）

**代码实际：** 并行运行 01/02/03 → 接口校验 → 护栏 → L1-SEM → 聚合终态

**偏差：** L1-02 定义要求 L1-01 先通过；代码已解耦（`l101_passed` deprecated）。
