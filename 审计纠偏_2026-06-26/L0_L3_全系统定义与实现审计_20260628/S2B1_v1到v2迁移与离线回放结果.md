## 3. 对象编号规则

路径：`对象编号.py`

格式：`{前缀}-{pipeline_run_id}-{序号:04d}`

| 对象 | 前缀 |
|------|------|
| L1 发现 | L1F |
| L1 失败包 | L1P |

- 同输入同顺序 → 同编号；无 UUID、无哈希
- 迁移上下文保存旧对象到新编号映射

## 4. 显式迁移规则

路径：`v1到v2迁移.py`

| 层 | v1 来源 | v2 目标 | 关键策略 |
|----|---------|---------|----------|
| L1 | 失败包 | L1发现项 + L1失败包 v2 | 问题域映射表；未知 → `L1_DOMAIN_MAPPING_MISSING` |
| L1.5 | 路由报告 | L1_5路由决策 v2 | `primary_failure` 解析为引用；route_id 来自 `L1.5_路由规则.json` |
| L2 | 修复单/报告 | L2修复单 + L2报告 v2 | 修复规则/动作/产物拆分；混合语义 → `L2_REPAIR_SEMANTICS_AMBIGUOUS` |
| L3 | 任务包/执行结果 | L3任务包 + L3执行结果 v2 | 执行模式与状态分离；候选产物须显式来源 |

歧义失败策略：必填缺失、语义不明、引用不闭合、Schema 不通过 → `FAILED`，不得塞入备注后继续。
