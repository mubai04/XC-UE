## 5. 实现符合性证据标准

判定「已实现」须同时满足：

```text
正式定义存在
+ 正式入口可达
+ 数据模型存在
+ 核心领域逻辑存在
+ 领域校验存在
+ 输出产物存在
+ 回流接口存在
+ 测试验证真实行为
```

以下**不能**判为已实现：仅有文件名、prompt、mock、Schema、确定性提取、夹具、状态声明或 API 成功返回 JSON。

## 6. 审计方法

1. **先治理后层级：** 真源权限 → L0 → L1 → L1.5 → L2 → L3 → 跨层契约 → 测试证据

3. **云盘副本核对：** OneDrive 下同名文件路径与内容分叉记录
4. **无证据标注：** `SOURCE_AUTHORITY_UNRESOLVED` / `NOT_AUDITABLE`
