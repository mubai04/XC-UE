# 运行记录管理规则

> XC-UE 本地运行产物治理规则（P1B-2 确立）

## 基本原则

1. **`运行记录/` 是本地执行生成目录**，用于测试、mock、smoke 及流水线中间产物。
2. **默认不纳入 Git**。新增 `.gitignore` 规则：`/运行记录/*`（保留 `运行记录/README.md`）。
3. **mock、pytest、smoke 和临时流水线输出不提交**。
4. **真实 API 原始证据**必须迁入正式结果目录（如 `tests/fixtures/*/results/`）。
5. **人工评分和人工裁决**必须迁入正式评审目录，不得仅存在于 `运行记录/`。
6. **审计引用的关键证据**必须进入 `审计纠偏_2026-06-26/` 或对应审计归档目录。
7. **状态生成文件**（如 AUDIT_BASELINE、RUNTIME_SNAPSHOT）由正式脚本重新生成，不作为永久真源。
8. **不得把运行记录目录当作项目真源**。
9. **删除本地运行记录不得影响正式证据**——正式证据必须在其他目录有副本或已迁出归档。
10. **每个新增评估脚本必须声明永久结果目录**，并在 README 或脚本头部注明。

## 与 Git 的关系

- 已被 Git 跟踪的历史 `运行记录/` 文件，经 P1B-2 裁决后可停止跟踪（专用清理提交，不重写历史）。
- 生产代码输出到 `运行记录/` **不等于**该目录必须被 Git 跟踪。
