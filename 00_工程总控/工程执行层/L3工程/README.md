# L3工程

本目录把 `50_L3_执行协议层/` 的 Markdown 协议转换为可运行的任务规划校验工程。

L3工程不判断正文质量，不替 L1/L2/L1.5 裁决。默认 `L3` 入口只生成 `TASK_PLANNING_ONLY` 任务包，并检查任务是否满足文件权限、目标引用、任务依赖、约束和复验入口要求。

`L3_PATCH` 是受限的 B2 运行时执行线：它只允许在人工审批明确通过、补丁内容冻结、哈希校验通过、目标仍为 `TP-001`、策略仍为 `L2-01` 的单一确定性 `REPLACE`/`APPEND`、且候选正文已经通过 L1 复验后，对正式正文执行原子应用或自动回滚。该运行时正式应用不属于 Codex 手工改写正文，也不允许调用模型或 API。

## 文件对应关系

| 协议文件 | 工程文件 | 职责 |
|---|---|---|
| L3-00_执行协议总表 | 协议解析.py / 任务单校验.py | 校验任务规划输入输出、状态机、权限矩阵 |
| L3-01_Cursor文件操作协议 | 文件操作校验.py | 校验目标文件、禁止修改文件、真源权限 |
| L3-02_正文生成与改写任务协议 | 正文任务校验.py | 校验正文任务规划必须有 IR、目标引用、修复方向 |
| L3-03_验收回填协议 | 验收回填校验.py | 校验复验入口 |
| L3-04_版本与回滚协议 | 版本回滚校验.py | 校验 TASK_PLANNING_ONLY 不要求正文备份 |
| L3-05_日志记录协议 | 日志记录校验.py | 校验任务断点记录字段 |
| L3-06_IR输入映射协议 | IR输入映射校验.py | 按 L2 模块映射 IR 输入 |
| L3-07_ProjectHarness运行协议 | ProjectHarness运行校验.py | 生成测试项目候选目标 |
| L3-99_执行禁止项 | L3禁止项检查.py | 检查删除、覆盖正式正文、越权判断等阻断项 |
| L3_PATCH 受限执行线 | L3补丁执行入口.py / L3补丁执行器.py | 生成候选、等待审批、正式应用与自动回滚 |

## 运行

```powershell
python "00_工程总控\工程执行层\统一运行入口.py" --target L3 --run-id L3_RUN-手动编号
```

指定 L2 报告：

```powershell
python "00_工程总控\工程执行层\统一运行入口.py" --target L3 --l2-report "00_工程总控\工程执行层\L2工程\reports\L2_RUN-20260622-003.json"
```

## 输出

报告输出到：

```text
00_工程总控/工程执行层/L3工程/reports/
```

输出内容包括：

- L3 任务规划单
- 协议校验结果
- 任务包文件
- 分项任务
- 任务依赖 / 约束 / 目标文件引用 / 复验入口
- 阻断任务
