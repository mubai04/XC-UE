# L2工程

本目录把 `40_L2_正式能力层/` 的 Markdown 能力标准转换为可运行的接口判断与修复单工程。

L2工程不检测文件壳，不直接写正文，不替 L1.5 裁决主失败；它接收 L1 failure packet，依据 L2-99 判断能进入哪个 L2 模块，并生成修复单。

## 文件对应关系

| 标准文件 | 工程文件 | 职责 |
|---|---|---|
| L2-00_正式能力层定义 | 能力标准解析.py | 校验 L2 标准关键章节 |
| L2-01_叙事结构能力 | L2_01_叙事结构能力.py | 生成叙事结构修复单 |
| L2-02_文风语言能力 | L2_02_文风语言能力.py | 生成文风语言修复单 |
| L2-03_角色心理能力 | L2_03_角色心理能力.py | 生成角色心理修复单 |
| L2-04_创意设定能力 | L2_04_创意设定能力.py | 生成创意设定修复单 |
| L2-05_市场体验能力 | L2_05_市场体验能力.py | 生成市场体验修复单 |
| L2-06_系统一致性能力 | L2_06_系统一致性能力.py | 生成系统一致性修复单 |
| L2-99_能力层接口总表 | L2_99_接口判断.py | 判断接口候选模块、越界、回流 |

## 运行

```powershell
python "00_工程总控\工程执行层\统一运行入口.py" --target L2 --run-id L2_RUN-手动编号
```

指定 failure packet：

```powershell
python "00_工程总控\工程执行层\统一运行入口.py" --target L2 --failure-packet "00_工程总控\工程执行层\L1工程\reports\L1_RUN-20260622-002_failure_packet.json"
```

## 输出

报告输出到：

```text
00_工程总控/工程执行层/L2工程/reports/
```

输出内容包括：

- L2-99 接口判断
- L2 修复单
- 阻断 / 回 L1.5 / 非 L2 出口

## 边界

- 不写正文。
- 不替 L1.5 派单裁决。
- 不把 L3、外部运营、文件落盘问题伪装成 L2。
- 不覆盖 Markdown 真源。
