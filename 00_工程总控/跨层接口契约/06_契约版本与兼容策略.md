# 06 契约版本与兼容策略

## 1. 版本线

| 线 | 位置 | 状态 |
|----|------|------|
| v1 运行时 | `结构定义/*.json`（非跨层契约目录） | **当前生产** |
| v2 冻结候选 | `结构定义/跨层契约/*_v2.json` | S2A 冻结，未接入运行时 |

## 2. S2A 边界

1. v1 保持当前运行，**内容不得被 S2A 修改**。
2. v2 是冻结候选契约，独立 `$id`。
3. S2A **不允许**运行时同时写 v1 和 v2。
4. 禁止静默 fallback；缺字段必须失败。
5. 旧历史文件只读。
6. 旧输入须经**显式迁移器**（S2B）转换。
7. 公共定义通过 `$ref` 引用 `公共引用结构_v1.json`，禁止复制。
8. 每个 Schema `additionalProperties: false`（关键对象）。

## 3. S2B 迁移要求（尚未开始）

- 选择明确迁移边界（读 v1 写 v2 或双写过渡期）。
- 实现 `迁移器`：v1 失败包 → v2 发现项/失败包等。
- 更新 L1/L1.5/L2/L3 生产代码消费 v2。
- 全量回归 + 真实 API 评测（若纳入 S2B 范围）。
