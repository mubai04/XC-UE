# 01 字段命名与语义规范

## L1 发现项

| 字段 | 语义 |
|------|------|
| L1发现编号 | 唯一 ID |
| L1问题域 | 粗粒度领域（叙事/文风/角色/设定/读者体验/故事一致性/工程技术/语义审计/发布准备） |
| L1失败类型 | L1 可观察失败标签；**不得**作为跨层公共字段名 |
| 候选模块提示 | 非权威提示；L1.5 `目标模块` 才是权威 |
| 修复提示 | 高层修复提示；不是 L2 动作 |
| blocking / routeable | 独立布尔；见 S1B |

## L1.5 路由决策

| 字段 | 语义 |
|------|------|
| 主发现引用 | 引用 `L1发现编号`；禁止 `primary_failure` 对象 |
| 目标模块 | **仅 L1.5 有权确定** |
| 修复产物类型 | 产物类别；不等于修复动作 |
| 路由状态 | L1.5 专用枚举 |

## L2 修复单

| 字段 | 语义 |
|------|------|
| 模块内主问题 | 已路由模块内部子问题；不是 L1 `L1失败类型` |
| 模块内次级问题 | 不得改变 L1.5 目标模块 |
| 修复规则 | 约束 |
| 修复动作 | 可执行步骤 |
| 修复产物类型 | 产物类别 |
| 重路由请求 | 只能请求回 L1.5；禁止指定新目标模块 |

## L3

| 字段 | 语义 |
|------|------|
| 执行模式 | `TASK_PLANNING_ONLY` / `CANDIDATE_GENERATION` |
| 执行状态 | 与执行模式分离 |
| 候选产物列表 | 正式字段；每项引用修复单编号 |

## 废止（v2 中禁止作为跨层字段）

- `status`（无层级）
- `最终状态`（无层级）
- `rule_source`（含混）
- `primary_failure`（对象型主失败）
- `失败类型`（无层级）
- `修复方向`（含混）
- `主失败类型`（L2 旧名）

## 规则来源拆分

| 字段 | 允许内容 |
|------|----------|
| 路由规则来源 | 仅 `30_L1.5_路由矩阵层/L1.5_路由规则.json` |
| 能力规则来源 | 仅 `40_L2_正式能力层/**/ability_rules.json` |
| 执行协议来源 | 仅 L3 protocol 规则 |
| 证据规则来源 | 证据协议文件 |

L2 旧 `routes.json` **不得**出现在路由规则来源。
