# 03 状态机边界

v2 禁止无层级 `status`、`最终状态`、`处理状态`。

## L1 顶层状态（L1顶层状态）

```text
SCREENING_PASS      — 无内容修复流水线问题
SCREENING_REVIEW    — 需人工复核或可选修复
SCREENING_REJECT    — 必须修复或护栏失败
AUDIT_BLOCKED       — 无法形成内容结论
```

对象：`L1失败包结构_v2.json`

## L1.5 路由状态（路由状态）

```text
ROUTED              — 已确定目标模块（须携带目标模块）
RETURN_TO_L1        — 回 L1 复验
INPUT_REQUIRED      — 需补输入
MANUAL_REVIEW       — 人工复核
BLOCKED_TECHNICAL   — 工程技术阻断
```

对象：`L1_5路由决策结构_v2.json`  
非 `ROUTED` 时 `目标模块` 必须为空。

## L2 修复单状态（修复单状态）

```text
READY_FOR_L3
INPUT_REQUIRED
RETURN_TO_L1_5
MANUAL_REVIEW
AUDIT_BLOCKED
```

对象：`L2修复单结构_v2.json`

L2 报告另有 `L2报告状态`：`COMPLETED` / `PARTIAL_BLOCKED` / `MODEL_BLOCKED` / `SCHEMA_INVALID` / `BLOCKED`

## L3 执行状态（执行状态）

```text
PLANNED
EXECUTING
CANDIDATE_WRITTEN
BLOCKED
FAILED
COMPLETED
```

对象：`L3执行任务包结构_v2.json`、`L3执行结果结构_v2.json`

## L3 执行模式（执行模式）— 与执行状态分离

```text
TASK_PLANNING_ONLY
CANDIDATE_GENERATION
```

禁止用执行模式值填充执行状态。

## 禁止混用

| 错误用法 | 正确字段 |
|----------|----------|
| L1.5 final_status | 路由状态 |
| L2 修复单.最终状态 | 修复单状态 |
| L3 status = TASK_PLANNING_ONLY | 执行模式 + 执行状态 |
| 失败包 status | L1顶层状态 |
