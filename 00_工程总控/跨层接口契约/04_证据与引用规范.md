# 04 证据与引用规范

## 1. 原则

- 禁止用单个自由文本 `证据` 字段承载全部来源。
- 所有 v2 证据使用 `公共引用结构_v1.json` 中的 `证据引用`。
- L2 诊断证据、L3 验收均通过 `证据引用` 数组。

## 2. 证据引用必填字段

| 字段 | 说明 |
|------|------|
| 证据编号 | 唯一 ID |
| 来源类型 | 见下表 |
| 来源路径 | 文件或逻辑路径 |
| 段落编号 | 可空 |
| 行号范围 | 起始行、结束行 |
| 逐字摘句 | 原文摘录 |
| 证据用途 | SCREENING / ROUTING / DIAGNOSIS / REPAIR_CONSTRAINT / ACCEPTANCE / AUDIT |

## 3. 来源类型枚举

| 值 | 含义 |
|----|------|
| CHAPTER | 当前章节正文 |
| PRIOR_CHAPTER | 前序章节 |
| IR | 中间表示文件 |
| PROJECT_RULE | 项目规则 |
| FAILURE_EVIDENCE | 上游失败包证据 |
| RUNTIME_STATE | 运行状态快照 |

## 4. 上游对象引用

用于跨层编号关联：

```json
{
  "对象类型": "L1发现项",
  "对象编号": "L1F-DEMO-001",
  "显示摘要": "文风重复（非权威）",
  "非权威说明": "显示摘要不是真源字段；引用编号才是关联真源"
}
```

## 5. 回流闸门引用

统一使用 `回流闸门引用`，含 `回流闸门` 枚举与可选 `复验说明`。

## 6. 规则来源（拆分 rule_source）

| 字段 | 路径约束 |
|------|----------|
| 路由规则来源 | 固定 `30_L1.5_路由矩阵层/L1.5_路由规则.json` |
| 能力规则来源 | `40_L2_正式能力层/**/ability_rules.json` |
| 执行协议来源 | `50_L3_*/protocol*` |
| 证据规则来源 | 证据协议文件路径 |

L2 废弃 `routes.json` 不得写入路由规则来源。
